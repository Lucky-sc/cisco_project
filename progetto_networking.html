Rete Campus.

01. Router 0: Cuore della Rete Ovest

Il Router 0 gestisce la porzione sinistra della topologia. È un nodo critico perché possiede il maggior numero di interfacce attive, garantendo la connessione locale e la ridondanza geografica.

  • VLAN 10 & 20: Suddivisione dei PC in gruppi virtuali per isolare i reparti, evitando conflitti di dati e loop.
  • Default Gateway: Le interfacce Fa0/0 e Fa1/0 collegano i PC locali agendo come uscita per le reti 192.168.1.x e 2.x.
  • Ponti Seriali: I link Se2/0 (200.100.20.1) e Se3/0 (200.100.40.1) permettono il dialogo diretto con gli altri uffici. 
interface Serial2/0
 ip address 200.100.20.1 255.255.255.0
!
interface Serial3/0
 ip address 200.100.40.1 255.255.255.0

02. Router 1: Nodo Centrale e Sicurezza

Il Router 1 funge da centro nevralgico per i servizi di sicurezza. Coordina l'accesso alla rete tramite il server RADIUS e gestisce diverse sottoreti locali per PC e amministrazione.

  • Interfaccia Fa1/1: Punto di interconnessione Campus configurato con IP 192.168.7.3.
  • Server RADIUS: Posizionato sull'interfaccia Fa0/5 dello Switch 3, gestisce l'autenticazione centralizzata AAA.
  • PC ADMIN (PC12): Configurato sulla porta Fa0/7 per la manutenzione dell'infrastruttura.
// Configurazione Router 1
interface FastEthernet1/1
 ip address 192.168.7.3 255.255.255.0
!
interface Serial2/2
 ip address 200.100.20.2 255.255.255.0

03. Router 2: Mobilità e DHCP

Il Router 2 gestisce l'infrastruttura wireless e l'assegnazione dinamica degli indirizzi tramite DHCP Relay.

  • Fa0/0 (LAN): IP 192.168.4.1 con IP-Helper puntato al server 192.168.4.10.
  • Fa1/1 (Interconnessione): Connessione alla dorsale 7.x con IP 192.168.7.2.
// CLI Router 2
interface FastEthernet1/1
 ip address 192.168.7.2 255.255.255.0
!
interface Serial2/0
 ip address 200.100.30.2 255.255.255.0

04. Politiche di Sicurezza & Firewall

Controllo Accessi e NAT

La configurazione utilizza il NAT dinamico per permettere alle reti interne (192.168.1.0 - 4.0) e alla rete di interconnessione (192.168.7.0) di navigare verso l'esterno.

  • • Protezione Entrante: Applicato un criterio "Deny All" di default. L'accesso è consentito solo per traffico HTTP/HTTPS verso l'host pubblico 192.168.6.10.
  • • Restrizioni Uscita: Bloccati esplicitamente i protocolli FTP e TFTP per prevenire esfiltrazioni; ammesso il traffico ICMP e IP generico.
  • • Routing Strategico: Rotte statiche configurate per dirigere il traffico verso i gateway 192.168.5.1 e 192.168.6.1.
// Configurazione Gigabit Ethernet
interface GigabitEthernet1/1
 ip address 192.168.6.2 255.255.255.0
 ip nat outside
!
interface GigabitEthernet1/2
 ip address 192.168.7.1 255.255.255.0
 ip nat inside
